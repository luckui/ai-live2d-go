# 说明书编写指南

本文件定义知识库条目的质量标准和编写规范。

---

## 知识库存放位置

知识库支持两种格式，AI 会自动扫描两处：

### 1. 传统格式（manual/）

```
live2d-pet/electron/manual/
  ├── dev/          ← 开发相关（TDD、调试、前端等）
  ├── ops/          ← 命令行、环境、部署
  ├── workflow/     ← 任务规划、通用工作流
  ├── browser/      ← 浏览器自动化操作流程
  ├── streaming/    ← 直播相关
  └── 说明书编写指南.md
```

每个 `.md` 文件 = 一个主题，支持可选的 YAML frontmatter。

### 2. Agent Skills 标准格式（skills/）

遵循 [agentskills.io](https://agentskills.io) 开放标准，与 Hermes Agent、VS Code Copilot 等工具兼容：

```
live2d-pet/electron/skills/
  ├── skill-name/
  │   └── SKILL.md    ← YAML frontmatter + Markdown 正文
  └── category/
      └── skill-name/
          └── SKILL.md
```

**推荐新建文档使用此格式。**

---

## 文件命名规范

**传统格式**：
- 使用中文命名，简洁直观：`命令行操作.md`、`网站登录流程.md`
- AI 支持模糊匹配，`read_manual("调试")` 可匹配 `dev/系统化调试工作流.md`

**Agent Skills 格式（SKILL.md）**：
- 文件夹名用小写 + 连字符：`bilibili-live/`、`browser-automation/`
- 必须包含 YAML frontmatter，`name` 和 `description` 字段必填

---

## SKILL.md 格式（推荐）

```markdown
---
name: skill-name          # 必须，小写 + 连字符，AI 用此名称匹配
description: 一句话描述   # 必须，≤100 字，显示在系统提示目录中
version: 1.0.0            # 可选
---

# Skill 标题

正文内容（Markdown 格式）。
只在 AI 调用 read_manual(topic="skill-name") 时才加载全文（渐进式披露）。
```

## 铁律（核心原则）

用一句话概括最重要的行为约束，不遵守就会出问题的规则。

例：`测试没红之前不写生产代码`、`调试不超过 3 步无效就换方向`

---

每步 = 2-5 分钟可完成的原子操作。

**执行**：
\```bash
# 实际命令（带真实参数）
\```
**预期结果**：应该看到什么输出

### 步骤 2：[下一步]
...

从实际经验中提取的错误模式：
- ⚠️ **陷阱名称**：问题描述 → 正确做法
- 🚫 **禁止**：明确列出不该做的事

如何确认执行成功：
1. 检查 XXX，预期结果是 YYY

## 交叉引用

相关说明书（有助于组合使用）：
- 相关主题名 → 什么场景配合使用

## 反模式清单

以下写法会降低说明书质量，必须避免：

| 反模式 | 问题 | 正确做法 |
|--------|------|----------|
| 只写概念不写命令 | AI 看了还是不知道怎么做 | 每步带可运行的命令示例 |
| 步骤太粗 | "配置环境"是伪步骤 | 拆到 2-5 分钟粒度 |
| 没有验证 | 做完不知道对不对 | 每个关键步骤有验证命令 |
| 无陷阱记录 | 重复踩坑 | 把出过的错误记到常见陷阱 |
| 虚构工具名 | 引用不存在的工具 | 只用实际可调用的工具名 |

---

## AI 何时会查阅说明书

AI 在以下情况下会主动调用 `read_manual`：
1. 准备调用工具完成任务时，先扫描说明书目录是否有相关主题
2. 工具执行失败后查阅正确用法
3. 用户说"翻一下手册"、"按说明书做"

**AI 不会在闲聊时查阅说明书，只在需要执行操作时才触发。**

---

## AI 何时应该创建/更新说明书

- 复杂任务完成后（5+ 工具调用、多次迭代、克服了错误）
- 用户纠正过的方法被验证有效后
- 使用说明书时发现步骤过时或缺漏 → 用 `manual_manage(action="patch")` 当场修正
