# 听觉系统（STT 语音识别）

## 概述

听觉系统让 Hiyori 能够"听到"用户的声音或系统音频，实时转写为文字。基于 faster-whisper 本地模型，无需联网。

## 环境变量

- `HIYORI_UV_EXE`：uv 可执行文件路径
- `HIYORI_DATA_DIR`：数据目录（%AppData%/Hiyori/）

## 工具：manage_hearing

### 操作速查

| 用户说 | action | 备注 |
|--------|--------|------|
| 安装听觉/语音识别 | install_and_start | 首次安装 |
| 开始听/帮我听 | start_listening | 需先安装，根据意图选 mode |
| 别听了/停 | stop_listening | |
| 听视频/听系统音频 | start_listening + source=system | P2 功能 |
| 你听到了什么 | get_transcript | 陪伴监听模式下读取缓存 |
| 清空听力缓存 | clear_transcript | |
| 语音识别状态 | status | |

### 听觉模式（mode 参数）

| 模式 | mode 值 | 说明 |
|------|---------|------|
| 语音输入 | dictation | 用户说完后自动识别为消息发送给你 |
| 陪伴监听 | passive | 只缓存转录，你需要用 get_transcript 读取（默认） |
| 总结模式 | summary | 缓存转录，停止听时自动发送全文请你总结 |

**选择建议：**
- 用户说"帮我听"、"开始听" → passive（默认，让你陪着听）
- 用户说"用语音输入"、"我说你打" → dictation
- 用户说"帮我记录/总结这段" → summary
- 用户说"听一下这个视频帮我总结" → summary + source=system

### 完整操作流程

```
1. install_and_start → 安装 Python 环境 + faster-whisper + 启动 STT 服务
2. start_listening (source=mic, mode=passive) → 开始监听麦克风
3. 转写结果实时显示在聊天界面（底部指示器+气泡流）
4. get_transcript → 读取用户说了什么（陪伴监听模式）
5. stop_listening → 停止监听（总结模式会自动发送全文）
6. stop → 停止 STT 服务（释放资源）
```

## 模型选择

| 模型 | 大小 | 速度 | 精度 | 适用场景 |
|------|------|------|------|----------|
| tiny | ~75MB | 最快 | 一般 | 快速测试 |
| base | ~150MB | 快 | 较好 | **推荐入门** |
| small | ~500MB | 中等 | 好 | 日常使用 |
| medium | ~1.5GB | 较慢 | 很好 | 高精度需求 |
| large-v3 | ~3GB | 慢 | 最好 | GPU 用户 |

**默认使用 base 模型**。用户可通过 model 参数指定。

## CUDA 检测

安装时自动检测 GPU：
- 有 CUDA GPU → float16 推理（更快）
- 无 GPU → CPU int8 推理（也能用，稍慢）

无需用户手动配置。

## 音频源

- **mic**：麦克风（默认），适合对话场景
- **system**：系统音频，适合听视频、听播放内容
- **both**：同时听麦克风和系统音频

## 技术细节

- STT 服务端口：9890（WebSocket）
- 协议：WebSocket 二进制帧 = PCM s16le 16kHz mono
- VAD：基于能量检测自动断句（0.8s 静音触发转写）
- 最短音频：0.5s，最长单段：30s

## 故障排查

1. **安装失败**：检查网络，pip 使用清华镜像
2. **启动超时**：首次加载模型需要下载，耐心等待
3. **转写不准**：尝试升级模型（base → small → medium）
4. **没有声音输入**：检查麦克风权限、系统音频设置
