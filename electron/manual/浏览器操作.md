# 浏览器操作

本手册记录 browser_* 工具的使用规范，**在需要进行浏览器操作时必须查阅**。

---

## 工具选择总则

| 用户意图 | 应调用工具 |
|---------|-----------|
| 打开/访问网页（URL/域名） | `browser_open` |
| 搜索（搜一下/查一下/关键词查询） | `browser_search` |
| 导航后判断页面/找链接/读内容 | `browser_read_page` |
| 点击页面按钮/链接 | `browser_click_smart` |
| 在输入框中输入文字 | `browser_type_smart` |

---

## 点击规范

使用 `browser_click_smart`，分两阶段：

**Phase 1（扫描）**：`browser_click_smart(text="搜索,search,查找")`
- 评分 ≥ 70 且唯一 → 直接点击
- 有歧义 → 返回带评分+class的候选列表，从候选列表中选 idx

**Phase 2（执行）**：`browser_click_smart(idx="编号")`

补充规则：
- `text` 参数强烈建议同时提供多同义词（逗号分隔）："搜索,search"、"登录,login"
- 候选列表的 `class` 字段是判断元素用途的重要依据（如 `nav-search-btn`、`submit-btn`）
- 仅当目标是 `<a>` 链接且能获取到 `href` 时，才用 `browser_open(href)` 直接导航（更可靠）
- ❌ 禁止直接调用 `browser_click` / `browser_js_click` / `browser_get_buttons`

---

## 搜索规范

- 用户表达"搜一下/查一下/关键词查询"时，优先调用 `browser_search`，**不要用 `browser_open`**
- `browser_open` 只用于导航（URL/域名/href）
- `browser_search(scope="auto")`：优先站内搜索，失败自动回退全网搜索
- 用户明确"全网搜" → `scope="web"`；明确"站内搜" → `scope="site"`

---

## 导航感知规范

`browser_open` 导航完成后，自动返回页面摘要（标题/URL/类型/链接），收到摘要后：

- 页面类型 = **搜索结果页** 且目标链接在列表中 → 立即用 `browser_click_smart` 点击目标链接
- 页面类型 = **内容详情页** → 可用 `browser_read_page(detail="full")` 读取正文
- 不确定页面内容 → `browser_read_page(detail="brief")` 确认

> ⚠️ 不要导航到一个页面就停止，要根据页面摘要继续决策下一步。

---

## 输入规范

使用 `browser_type_smart`，分两阶段：

**Phase 1（扫描）**：`browser_type_smart(description="输入框描述", value="内容")`
- 评分 ≥ 70 且唯一 → 直接执行，无需 Phase 2
- 存在歧义 → 返回带评分的候选列表，从中选择 idx

**Phase 2（执行）**：`browser_type_smart(idx="编号", value="内容")`

- `description` 填写输入框旁边的 label 文字、placeholder 或用途描述（如"用户名"、"搜索框"、"密码"）
- ❌ 禁止直接调用 `browser_type` / `browser_get_inputs` / `browser_type_rich`，这些由 `browser_type_smart` 内部自动处理
