# B站UP主查找工作流

## 概述
本工作流用于帮助用户快速定位B站平台指定用户名的UP主账号，适用于内容检索、竞品分析、UP主信息调研等场景。可高效获取目标UP主的粉丝数据、投稿内容、账号属性等核心信息，避免搜索路径错误导致的查询失败。

## 触发条件
何时应该使用此工作流：
- 用户明确提出需要搜索B站指定用户名的UP主信息、投稿内容的需求
- 需要完成B站UP主账号调研、内容采集类任务
- 前置条件：可正常访问B站官方网站，已获取待查找的UP主用户名

## 工具建议

### 1. 访问B站并搜索UP主
**场景**: 在B站搜索指定用户名 → 使用 `browser_open`：

```javascript
browser_open(query="bilibili.com/search?keyword=GeoLingua")
```

**说明**: 直接通过搜索URL访问B站搜索页面，`browser_open` 会自动等待页面加载并返回包含搜索结果的页面摘要。

**预期结果**: 返回搜索结果页面，包含"视频"、"番剧"、"用户"等分类标签，默认显示视频搜索结果。

### 2. 切换到用户搜索结果
**场景**: 从综合搜索切换到用户列表 → 使用 `browser_find`：

```javascript
browser_find(keyword="用户", category="button")
```

**说明**: 在搜索结果页面查找"用户"分类标签并点击，`browser_find` 会智能匹配按钮文本并执行点击操作。

**预期结果**: 页面刷新，显示匹配关键词的UP主账号列表，按粉丝数或相关度排序。

### 3. 查看页面状态
**场景**: 确认是否成功加载UP主列表 → 使用 `browser_screenshot`：

```javascript
browser_screenshot()
```

**说明**: 截取当前页面的屏幕截图，用于AI分析页面内容、定位目标UP主条目。

**预期结果**: 返回页面截图，可以识别出UP主的头像、用户名、粉丝数等信息。

### 4. 点击目标UP主
**场景**: 进入目标UP主的个人主页 → 使用 `browser_click`：

```javascript
browser_click(selector="text=GeoLingua")
```

**说明**: 使用Playwright选择器点击匹配的UP主用户名链接，跳转到个人主页。

**预期结果**: 成功进入UP主个人主页，显示粉丝数、投稿数、个人简介、视频列表等完整信息。

## 操作步骤
**每个步骤 = 2-5 分钟内可完成的原子操作**

### 步骤 1: 访问B站搜索页面
**目标**: 打开B站搜索指定用户名的结果页

**执行**:
```javascript
browser_open(query="bilibili.com/search?keyword=GeoLingua")
```

**预期结果**: 成功加载B站搜索结果页，页面显示包含"视频"、"番剧"、"用户"等分类标签。

### 步骤 2: 切换到用户搜索结果
**目标**: 筛选出匹配用户名的UP主账号列表

**执行**:
```javascript
browser_find(keyword="用户", category="button")
```

**预期结果**: 页面刷新展示所有匹配搜索关键词的UP主账号，按相关度排序。

### 步骤 3: 截图确认页面内容
**目标**: 确认UP主列表是否正确加载

**执行**:
```javascript
browser_screenshot()
```

**预期结果**: 返回页面截图，可以看到UP主的头像、用户名、粉丝数等信息。

### 步骤 4: 进入目标UP主个人主页
**目标**: 访问UP主完整账号信息和投稿内容

**执行**:
```javascript
browser_click(selector="text=GeoLingua")
```

**预期结果**: 成功加载UP主个人主页，可查看粉丝数、投稿数、个人简介、视频投稿列表等完整信息。

## 常见陷阱
从实际对话中提取的错误模式和规避方法：
- ⚠️ **未切换用户标签导致查询失败**: 默认B站搜索结果仅展示视频、专栏等内容，未切换到「用户」标签页会提示找不到对应UP主，解决方案：搜索完成后优先点击「用户」分类筛选账号结果。
- ⚠️ **用户名拼写错误导致无匹配结果**: 输入的用户名存在错别字、符号遗漏/多余等问题时，会出现无匹配UP主的情况，解决方案：先与用户确认用户名拼写准确性，确认后再执行搜索。
- ⚠️ **禁止**: 未切换用户标签的情况下直接告知用户无对应UP主；用户名拼写未确认的情况下反复执行无效搜索。
