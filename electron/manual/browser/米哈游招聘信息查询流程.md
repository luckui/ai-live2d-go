# 米哈游招聘信息查询流程

## 概述
本工作流用于查询米哈游官方发布的校招、社招、实习生类招聘岗位、投递规则、培养政策、招聘动态等权威信息。通过浏览器自动化工具访问官方招聘网站，等待页面完全加载（动态渲染），然后提取结构化数据。适用于用户有米哈游招聘相关信息查询需求的场景。

## 触发条件
何时应该使用此工作流：
- 用户明确提出查询米哈游招聘相关信息（含岗位列表、投递规则、培养政策、招聘进度等）
- 用户要求获取官方发布的权威招聘内容，未指定第三方信息来源
- 前置条件：浏览器工具可用，网络连接正常

## 工具建议

### 1. 访问招聘官网
**场景**: 打开米哈游官方招聘入口 → 使用 `browser_open`：

```javascript
browser_open(query="https://jobs.mihoyo.com")
```

**说明**: 直接访问官方域名，避免通过搜索引擎（可能进入仿冒网站）。页面为动态渲染，`browser_open` 会自动等待 DOM 加载完成并返回页面摘要（标题/URL/页面类型/主内容链接）。

**预期结果**: 返回页面摘要，包含"校园招聘"、"社会招聘"、"实习生招聘"导航链接。

### 2. 切换招聘专区
**场景**: 根据查询类型切换到对应专区（校招/社招/实习） → 使用 `browser_click_smart`：

```javascript
// 校园招聘
browser_click_smart(text="校园招聘,campus")

// 或社会招聘
browser_click_smart(text="社会招聘,social")
```

**说明**: `browser_click_smart` 会自动扫描页面按钮/链接，匹配评分 ≥ 70 的元素并点击。`text` 参数建议提供多个同义词（逗号分隔）以提高识别准确率。

**预期结果**: 页面 URL 变化（如 `/campus` 或 `/social`），内容区域更新为对应专区的岗位列表。

### 3. 读取岗位列表
**场景**: 提取岗位信息、投递规则、培养政策 → 使用 `browser_read_page`：

```javascript
browser_read_page(detail="full")
```

**参数说明**:
- `detail="brief"`: 仅返回标题/URL/页面类型/主内容链接（快速确认页面内容）
- `detail="full"`: 返回完整信息（标题/URL/大纲/链接/正文摘要/可交互元素）

**预期输出**: 结构化 Markdown 文本，包含：
- 页面大纲（H1~H3 标题层级）
- 岗位卡片信息（标题、地点、职责、要求、报名链接）
- 可交互元素（输入框、按钮的操作提示）

### 4. 截图确认（调试用）
**场景**: 当页面摘要不足以判断内容，或需要人工复核时 → 使用 `take_screenshot`：

```javascript
take_screenshot()
```

**说明**: 截取当前屏幕（包括浏览器窗口），返回图像给 AI 多模态分析。需要视觉模型支持。

**预期结果**: 返回 PNG 格式屏幕截图的 base64 编码，AI 可直接分析图像内容。

## 操作步骤

### 步骤 1: 访问官方招聘入口
**目标**: 打开米哈游官方招聘网站首页

**执行**:
```javascript
browser_open(query="https://jobs.mihoyo.com")
```

**预期结果**: `browser_open` 自动等待页面加载完成后返回页面摘要，包含：
- 页面标题（如"米哈游招聘官网"）
- URL: `https://jobs.mihoyo.com`
- 页面类型: 网站首页
- 主内容链接: 校园招聘、社会招聘、实习生招聘（导航链接）

**验证**: 检查返回的 URL 域名为 `mihoyo.com`（官方认证域名），主内容链接包含"校园招聘"等导航。

### 步骤 2: 切换到对应招聘专区
**目标**: 根据用户查询的招聘类型（校招/社招/实习），点击导航切换专区

**执行**:
```javascript
// 校园招聘
browser_click_smart(text="校园招聘,campus")

// 或社会招聘
browser_click_smart(text="社会招聘,social")

// 或实习生招聘
browser_click_smart(text="实习生招聘,intern")
```

**预期结果**: `browser_click_smart` 自动扫描页面，找到匹配"校园招聘"的导航链接并点击，页面内容区域切换为对应专区的岗位列表。

**验证**: 再次调用 `browser_read_page(detail="brief")` 确认 URL 变化（如 `/campus`）、页面标题更新。

### 步骤 3: 读取岗位列表
**目标**: 提取岗位列表、投递规则、培养政策等内容

**执行**:
```javascript
browser_read_page(detail="full")
```

**预期结果**: 返回完整页面摘要，包含：
- **页面大纲**（H1~H3 标题层级，如"技术类岗位"、"游戏策划岗位"）
- **主内容链接**（岗位详情链接，含岗位名称和URL）
- **正文摘要**（投递规则、培养政策、停止招聘公告等文字内容）
- **可交互元素**（如有搜索框、筛选按钮，会显示操作提示）

**验证**: 内容包含至少 3 个岗位信息（岗位标题 + 详情链接），或明确的"暂无岗位"提示。

### 步骤 5: 结构化返回用户
**目标**: 将提取的信息整理为清晰的分类结构返回给用户

**执行**:
```markdown

**执行**:

**预期结果**: 导航到岗位详情页，`browser_open` 返回详情页摘要，包含职责描述、任职要求等完整信息。

**验证**: 页面类型为"内容详情页"，正文摘要包含岗位详细要求。

### 步骤 5: 结构化返回用户
**目标**: 将提取的信息整理为清晰的分类结构返回给用户 `browser_open(query="https://jobs.mihoyo.com")`。

- ⚠️ **不要手动等待页面加载**: `browser_open` 和 `browser_click_smart` 已内置页面加载等待机制（DOM 解析 + 网络空闲 + 主内容区出现），**无需额外调用等待工具**。错误做法：调用不存在的 `wait_for_selector` 工具。

- ⚠️ **不要用 `browser_open` 搜索关键词**: 用户说"帮我搜一下米哈游招聘岗位"时，**不要** `browser_open(query="米哈游招聘岗位")`。**正确做法**: `browser_search(query="米哈游招聘岗位", scope="web")`（全网搜索）或先 `browser_open(query="https://jobs.mihoyo.com")` 再 `browser_search(scope="site")` 站内搜索。
