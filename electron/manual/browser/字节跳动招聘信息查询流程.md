# 字节跳动招聘信息查询工作流

## 概述
本工作流用于高效获取字节跳动官方发布的校招、实习、社招等全品类招聘信息，适用于求职人群筛选匹配自身需求的岗位，可避免非官方渠道的虚假、过时招聘信息，大幅提升求职信息收集效率。

## 触发条件
何时应该使用此工作流：
- 用户明确提出查询字节跳动招聘（校招/实习/社招）相关需求
- 需要汇总字节跳动最新在招岗位、招聘要求、工作地点等核心信息
- 前置条件：可正常访问公共互联网

## 工具建议
**CRITICAL**: 必须明确列出每个工具的使用场景和调用方式：

- **访问字节跳动官方招聘网站** → 使用 `browser_open`：
  ```javascript
  browser_open(query="jobs.bytedance.com")
  ```
  说明：直接访问字节跳动招聘官网，自动等待页面加载并返回页面摘要。

- **截图确认页面内容** → 使用 `browser_screenshot`：
  ```javascript
  browser_screenshot()
  ```
  说明：截取当前页面，用于确认招聘信息和筛选选项。

- **读取详细内容** → 使用 `browser_read_page`：
  ```javascript
  browser_read_page(detail="high")
  ```
  说明：提取页面完整内容，包含所有岗位信息、招聘要求等。

## 操作步骤
**每个步骤 = 2-5 分钟内可完成的原子操作**

### 步骤 1: 进入字节跳动官方招聘平台
**目标**: 访问官方渠道确保招聘信息真实有效
**执行**:
```javascript
browser_open(query="jobs.bytedance.com")
```
**预期结果**: 成功进入字节跳动招聘官网，返回页面摘要包含招聘类型导航。

### 步骤 2: 筛选对应招聘类型
**目标**: 定位到用户需求的招聘项目类别
**执行**:
1. 在官网顶部导航栏选择对应招聘板块：校招选「校园招聘」、实习选「实习生招聘」、社招选「社会招聘」
2. 若有指定专项招聘需求，直接选择对应招聘专项入口（如Seed大模型专项、筋斗云人才计划等）
**预期结果**: 进入对应招聘项目列表页，可看到当前开放的所有招聘批次、专项说明。

### 步骤 3: 多维度筛选岗位
**目标**: 缩小岗位范围，匹配用户的岗位类别、工作地点等需求
**执行**:
在页面筛选栏依次选择对应维度：
- 岗位大类：研发/产品/运营/设计/职能等
- 工作地点：北京/上海/深圳/杭州/海外等
- 其他要求：毕业时间、工作经验等
**预期结果**: 筛选后展示符合所有条件的在招岗位列表，包含岗位名称、招聘要求、工作地点、投递入口等信息。

### 步骤 4: 结构化汇总招聘信息
**目标**: 输出清晰易懂的有效招聘信息
**执行**:
提取核心信息按分类整理：
1. 开放招聘项目/专项列表
2. 符合条件的岗位大类及总在招岗位数
3. 热门岗位示例
4. 覆盖工作地点范围
**预期结果**: 得到无冗余、分类清晰的招聘信息汇总，可直接交付给用户。

## 常见陷阱
从实际对话中提取的错误模式和规避方法：
- ⚠️ **非官方渠道信息失真**: 第三方招聘平台发布的字节招聘信息可能过时、虚假，甚至存在诈骗风险，解决方案：必须优先访问字节跳动官方招聘域名`jobs.bytedance.com`，不要点击来路不明的招聘链接。
- ⚠️ **禁止**: 汇总信息时添加未在官网公开的招聘要求、内推渠道、截止时间等非官方信息，避免误导用户。

如何确认工作流执行成功：

2. **验证点 2**: 核对汇总的招聘项目、总岗位数，预期结果和官网实时展示的内容完全一致，无信息
