# 命令行操作

Windows 系统下常用命令行操作规范，适用于 run_command 工具。

**run_command 底层走 `cmd.exe /c`。规则：**
- ✅ 可以用 `powershell -NoProfile -Command "..."` 把 PowerShell 当子进程调用——这是合法的 cmd 命令
- ❌ 不能直接裸写 PowerShell 专属语法（`Get-ChildItem`、`$env:`、`|Select-Object` 等）作为独立 cmd 命令

---

## 磁盘与存储

### 查看所有磁盘总容量与可用空间（推荐，无乱码）
```cmd
powershell -NoProfile -Command "Get-CimInstance Win32_LogicalDisk | Select-Object DeviceID,@{N='总GB';E={[math]::Round($_.Size/1GB,1)}},@{N='可用GB';E={[math]::Round($_.FreeSpace/1GB,1)}} | ft -Auto"
```
输出示例：
```
DeviceID  总GB  可用GB
--------  ----  ------
C:         300    13.2
D:       631.2    62.1
```

> ⚠️ 禁止用 `dir C:\` 查磁盘空间：`dir` 的卷标表头是 GBK 编码，chcp 65001 之后仍然乱码；
> 禁止用 `dir C:\ | findstr "字节可用"`：同理，GBK 字节流里找不到 UTF-8 编码的汉字，返回退出码 1。

### dir 的合法用途：列文件（/b 裸格式，无表头，无乱码）
```cmd
dir "D:\目标目录" /b
```
```cmd
dir /s /b "%USERPROFILE%\Desktop\文件名.ext"
```
`/b` 只输出文件名，没有卷标/序列号等中文表头，不会乱码。

## 内存

### 查看物理内存总量和可用量（单位 MB）
```cmd
powershell -NoProfile -Command "Get-CimInstance Win32_OperatingSystem | Select-Object @{N='总内存MB';E={[math]::Round($_.TotalVisibleMemorySize/1KB)}},@{N='可用内存MB';E={[math]::Round($_.FreePhysicalMemory/1KB)}} | ft -Auto"
```
输出示例：`总内存MB  可用内存MB` / `63093     31944`

> ⚠️ 禁止用 `systeminfo | findstr "内存"` 查内存：`systeminfo` 输出 GBK 编码，但命令串本身是 UTF-8，findstr 匹配失败会返回退出码 1、无输出。

## 进程与性能

### 查看所有运行中的进程
```cmd
tasklist /fo table
```

### 按名称筛选进程
```cmd
tasklist /fi "imagename eq chrome.exe"
```

### 结束进程（谨慎使用）
```cmd
taskkill /f /im 进程名.exe
```

### 查看内存概况
```cmd
powershell -NoProfile -Command "Get-CimInstance Win32_OperatingSystem | Select-Object @{N='总内存MB';E={[math]::Round($_.TotalVisibleMemorySize/1KB)}},@{N='可用内存MB';E={[math]::Round($_.FreePhysicalMemory/1KB)}} | ft -Auto"
```
输出单位为 MB，包含总量和可用量。

## CPU 占用率

> 以下命令格式：`powershell -NoProfile -Command "..."` 是 **cmd 命令**（启动 powershell.exe 子进程），引号内是 PowerShell 语法。`$_` 等符号会被 cmd 原样传给 PowerShell，不会被 cmd 展开。
>
> ⚠️ **路径引号规则（重要）**：`-Command "..."` 外层已经用了双引号，内层路径**必须用单引号**，否则 cmd.exe 会截断外层引号导致路径丢失、命令作用在当前目录：
> - ✅ 正确：`powershell -NoProfile -Command "Get-ChildItem 'D:\' -Directory"`
> - ❌ 错误：`powershell -NoProfile -Command "Get-ChildItem \"D:\\" -Directory"`（`\"` 在 cmd 里无效）

### 列出指定目录下的所有子文件夹（推荐写法）
```cmd
powershell -NoProfile -Command "Get-ChildItem 'D:\' -Directory | Select-Object Name"
```
```cmd
powershell -NoProfile -Command "Get-ChildItem 'C:\Users' -Directory | Select-Object Name"
```
路径用**单引号**包裹，避免与 `-Command "..."` 的外层双引号冲突。

### 列出指定目录下的所有文件（包含子目录）
```cmd
powershell -NoProfile -Command "Get-ChildItem 'D:\目录' -Recurse -File | Select-Object FullName"
```

### 查看当前总 CPU 占用率（实时百分比，0-100）
```cmd
powershell -NoProfile -Command "Get-CimInstance Win32_PerfFormattedData_PerfOS_Processor | Where-Object Name -eq _Total | Select-Object -ExpandProperty PercentProcessorTime"
```
输出示例：`11`（表示当前 CPU 总占用为 11%）

### 查看 CPU 占用率最高的进程（实时，前 10 名）
```cmd
powershell -NoProfile -Command "Get-CimInstance Win32_PerfFormattedData_PerfProc_Process | Where-Object {$_.Name -ne '_Total' -and $_.Name -ne 'Idle'} | Sort-Object PercentProcessorTime -Desc | Select-Object -First 10 Name,IDProcess,PercentProcessorTime | ft -Auto"
```
输出字段：`Name`（进程名）、`IDProcess`（PID）、`PercentProcessorTime`（CPU 占用率 %）

### 查看指定进程的 CPU 占用率
```cmd
powershell -NoProfile -Command "Get-CimInstance Win32_PerfFormattedData_PerfProc_Process | Where-Object Name -like chrome* | Sort-Object PercentProcessorTime -Desc | Select-Object Name,IDProcess,PercentProcessorTime | ft -Auto"
```
把 `chrome` 替换为要查询的进程名，支持 `*` 通配符（如 `electron*`、`Code*`）

### 查看 CPU 硬件信息（型号、核心数、频率）
```cmd
powershell -NoProfile -Command "Get-CimInstance Win32_Processor | Select-Object Name,NumberOfCores,NumberOfLogicalProcessors,MaxClockSpeed | ft -Auto"
```
- `NumberOfCores`：物理核心数
- `NumberOfLogicalProcessors`：逻辑线程数（超线程后）
- `MaxClockSpeed`：最大频率，单位 MHz（÷1000 = GHz）

### 查看进程累计 CPU 时间排名（启动以来消耗最多 CPU 的进程）
```cmd
powershell -NoProfile -Command "Get-Process | Sort-Object CPU -Descending | Select-Object -First 10 Name,Id,@{N='CPU累计秒';E={[math]::Round($_.CPU,1)}},@{N='内存MB';E={[math]::Round($_.WorkingSet/1MB,1)}} | ft -Auto"
```
> `CPU累计秒` 是进程启动以来消耗的总 CPU 时间（秒），**不是实时占用率**，可用于判断哪个进程历史上使用 CPU 最多。

### 用原生 cmd 查看单个进程的 CPU 累计时间
```cmd
tasklist /v /fo table /fi "imagename eq 进程名.exe"
```
输出中的 `CPU Time` 列格式为 `时:分:秒`，是自进程启动以来的累计 CPU 时间。

## 文件操作

### ⚠️ 写文件前必须先查路径，禁止猜测
用户说"放到桌面"、"放到文档"时，**必须先用 run_command 查出真实路径，再传给 write_file**，不能假设路径。

查桌面路径：
```cmd
echo %USERPROFILE%\Desktop
```
查文档路径：
```cmd
echo %USERPROFILE%\Documents
```
查下载路径：
```cmd
echo %USERPROFILE%\Downloads
```

---

### 获取常用文件夹路径
```cmd
echo %USERPROFILE%\Desktop
echo %USERPROFILE%\Documents
echo %USERPROFILE%\Downloads
```

### 查找文件（找不到时 dir 会输出"找不到文件"并返回退出码1，结果明确）
```cmd
dir /s /b "D:\目录\文件名.ext"
```

示例——在桌面查找 Layout.jpg：
```cmd
dir /s /b "%USERPROFILE%\Desktop\Layout.jpg"
```

### 列出目录下所有文件
```cmd
dir "D:\目录" /b
```

### 查看文本文件内容
```cmd
type "D:\path\to\file.txt"
```

### 复制文件夹
```cmd
robocopy D:\source D:\dest /s /e
```

### 删除文件
```cmd
del "D:\path\to\file.txt"
```

## 注意事项

- **✅ 可以**用 `powershell -NoProfile -Command "..."` 调用 PowerShell 查 CPU、进程、硬件信息——这是合法的 cmd 调用方式，引号内写 PowerShell 语法。
- **❌ 禁止**直接把 PowerShell 语法（`Get-ChildItem`、`$env:`、`[Environment]::`、`Select-Object` 等）当成独立的 cmd 命令，cmd.exe 无法识别，会返回退出码1。
- **❌ 禁止使用 `wmic`**，Windows 11 新版已移除，会报"不是内部或外部命令"；改用 `Get-CimInstance`（通过 `powershell -Command`）。
- **❌ 禁止用 `dir C:\`（不带 `/b`）查磁盘信息**，也禁止 `dir C:\ | findstr "字节可用"`：`dir` 的卷标/表头是 GBK 输出，chcp 65001 后仍乱码，findstr 中文匹配失败退出码 1。`dir` 只在加了 `/b` 裸格式时用于列文件名。
- **❌ 禁止用 `systeminfo | findstr "内存"`**：`systeminfo` 输出 GBK，findstr 搜 UTF-8 汉字字节不匹配，无输出退出码 1。
- 磁盘空间用 `powershell -NoProfile -Command "Get-CimInstance Win32_LogicalDisk | Select-Object DeviceID,Size,FreeSpace | ft -Auto"` 查看，内存用 `powershell -NoProfile -Command "Get-CimInstance Win32_OperatingSystem | Select-Object TotalVisibleMemorySize,FreePhysicalMemory | ft -Auto"`，CPU 占用率用 `powershell -NoProfile -Command "Get-CimInstance Win32_PerfFormattedData_PerfOS_Processor | Where-Object Name -eq _Total | Select-Object -ExpandProperty PercentProcessorTime"`。
- 路径中有空格时必须用双引号包裹：`"C:\Program Files\"`。
- `%USERPROFILE%` 是 cmd 的环境变量写法，等价于 PowerShell 的 `$env:USERPROFILE`。
- `dir /s /b` 找不到文件时会输出"找不到文件"并以退出码1退出，无需额外处理即可判断结果。
- `powershell -Command` 中的 `$_`、`$_.Name` 等变量会被原样传给 PowerShell，cmd 不会展开，可以放心使用。
- **`-Command "..."` 内层的路径/字符串必须用单引号**：`'D:\path'` 而不是 `\"D:\\path\"`，因为 cmd.exe 不识别 `\"` 转义，嵌套双引号会导致外层提前闭合，路径参数丢失。
